Variables & Classes - summaries

This file describes the main classes in the project and their key variables/methods.

1) User
- Purpose: Base account type. Stores username/password and common login behavior.
- Key fields: username, password
- Key methods: getUsername(), verifyPassword(password)
- Notes: Abstract base for `Customer` and `Admin`.

2) Customer
- Purpose: Represents a regular user who buys products.
- Key fields: personalCart (the customer's `Cart`), fullName, age, address, paymentMethod, cardNumber, orderHistory
- Key methods: getPersonalCart(), checkout()
- Notes: Inherits from `User` and holds its own `Cart` and order history.

3) Admin
- Purpose: Represents an admin user who manages products.
- Key fields: (none unique besides inherited ones)
- Key methods: updateStock(product, newStock), updatePrice(product, newPrice), addNewProduct(product)
- Notes: Inherits from `User` and can modify inventory.

4) Product
- Purpose: A product sold by the shop.
- Key fields: id, name, price, stockQuantity, specs (key/value map), isHotSale, category
- Key methods: reduceStock(amount)
- Notes: Serialized/storable product model used across views and cart.

5) Cart
- Purpose: Holds items a user wants to purchase.
- Key fields: items (list of `CartItem`)
- Key methods: addItem(product, qty), removeItem(product), calculateTotal(), clearCart(), getItems()
- Notes: Used by both `Customer.personalCart` and `Core.guestCart`.

6) CartItem
- Purpose: One line in a cart (product + quantity).
- Key fields: product, quantity
- Key methods: getTotalPrice(), increaseQuantity(amount)

7) PurchaseOrder
- Purpose: A completed order record.
- Key fields: orderId, customer, items (list of `CartItem`), totalAmount, orderDate
- Key methods: getTotal()
- Notes: Represents purchases saved to order history.

8) Core
- Purpose: Main controller / application state (singleton-like).
- Key fields: instance, usersDatabase (list of `User`), inventory (list of `Product`), loggedInUser, guestCart, hotProducts
- Key methods: getInstance(), authenticate(username,password), registerCustomer(...), processCheckout(), findProductById(id)
- Notes: Central place where views ask for data or perform actions.

9) DataManager
- Purpose: Handle loading/saving data from files.
- Key fields: dataFile (or files)
- Key methods: loadUsers(), saveUsers(list), loadProducts(), saveProducts(list)
- Notes: Used by `Core` to persist users and products.

10) GUI_MainFrame
- Purpose: Main application window and navigation container.
- Key fields: layout (CardLayout), core, mainPanel, homePanel, cartPanel, adminPanel, loginPanel, customerPanel, productPanel
- Key methods: GUI_MainFrame(), showHome(), showCart(), showAdmin(), showLogin(), showCustomer()
- Notes: Switches between view panels (Home, Cart, Admin, etc.).

11) HomeView
- Purpose: The landing/home screen showing hot products and categories.
- Key fields: homePanel, hotProductsPanel, core, mainFrame
- Key methods: HomeView(mainFrame), refreshHome()

12) ProductView
- Purpose: Displays details of a single product.
- Key fields: productPanel, core, mainFrame
- Key methods: ProductView(mainFrame), displayProduct(product)

13) PhoneView, SmartHomeView, OtherView
- Purpose: Category-specific product listing pages.
- Key fields: their panel(s)
- Key methods: constructors that receive `GUI_MainFrame`

14) SearchView
- Purpose: Lets users search products.
- Key fields: searchPanel, core
- Key methods: SearchView(mainFrame), search(query) -> list of products

15) CartView
- Purpose: UI that shows the contents of a cart and the total.
- Key fields: cartPanel, core, itemPanel, totalLabel, mainFrame
- Key methods: CartView(mainFrame), refreshCart()

16) AdminView
- Purpose: UI for admin actions (add/edit products, view orders).
- Key fields: adminPanel, core, mainFrame
- Key methods: AdminView(mainFrame)

17) AdminOrderHistoryDialog
- Purpose: Dialog that shows a customer's order history to an admin.
- Key fields: core, targetCustomer
- Key methods: AdminOrderHistoryDialog(parentFrame)

18) AddProduct, EditProduct
- Purpose: Dialogs/forms for creating or editing products.
- Key fields: related product (for edit), admin view reference
- Key methods: constructors to open dialogs, form handlers to save changes

19) ShowEditProfile
- Purpose: Dialog for customers to view/edit their profile.
- Key fields: customer
- Key methods: constructor that loads customer data and saves edits

20) LoginView
- Purpose: Login / register UI.
- Key fields: loginPanel, usernameField, passwordField, mainFrame
- Key methods: LoginView(mainFrame), login handling that calls `Core.authenticate()`

21) CustomerView
- Purpose: Main view for logged-in customer (profile, history, etc.).
- Key fields: customerPanel, core, mainFrame
- Key methods: CustomerView(mainFrame)

22) Main
- Purpose: Application entry point.
- Key methods: main(String[] args) — creates `GUI_MainFrame` and starts the app.

Common relationships (quick summary):
- `Customer` and `Admin` extend `User`.
- `Customer` has a `Cart` (composition). `Cart` contains `CartItem` objects, and each `CartItem` links to a `Product`.
- `Core` holds application data: lists of users and products, current user, guest cart, and order records.
- Most UI classes hold a reference to `Core` and to `GUI_MainFrame` so they can request data and switch views.
